# Notification History Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add in-memory notification history to cooee, queryable via a new `cooee history` CLI subcommand.
**Architecture:** History is stored in `AppState` as a `VecDeque<Notification>` bounded by `HistoryConfig::max_entries`; each notification is recorded after construction (after the DND `full` early-return) in `dbus.rs`; the `cooee history` CLI command sends `Command::History { count }` over the Unix socket and prints results formatted by timestamp, app name, urgency, and summary.
**Tech Stack:** `chrono 0.x` (serde feature for ISO 8601 timestamps), existing `serde_json`, `VecDeque` from std.

---

## Prerequisites

This plan assumes the **polish plan** has already been applied. Concretely:

- `Notification` struct already has a `replaces_id: u32` field.
- `Notification::new()` already accepts 12 parameters: `id, app_name, app_icon, summary, body, actions, urgency, expire_timeout, image_data: Option<ImageData>, image_path: Option<String>, replaces_id: u32` (plus whatever else the polish plan adds).
- `SharedConfig = Arc<RwLock<Config>>` exists in `config.rs`.
- The config already uses the `default_val!` macro pattern.

All code in this plan is written against those post-polish-plan signatures.

- [ ] **3.4 — Add `push_history` method**

  Inside the `impl AppState` block, add:

  ```rust
  /// Append or replace in-place; trims oldest entry when at capacity. No-op if history_max == 0.
  pub fn push_history(&mut self, n: Notification) {
      if self.history_max == 0 {
          return;
      }
      if n.replaces_id > 0 {
          if let Some(pos) = self.history.iter().position(|e| e.id == n.replaces_id) {
              self.history[pos] = n;
              return;
          }
      }
      if self.history.len() >= self.history_max {
          self.history.pop_front();
      }
      self.history.push_back(n);
  }
  ```

- [ ] **3.5 — Add `get_history` method**

  Inside the `impl AppState` block, add:

  ```rust
  /// Returns up to `count` most recent entries, newest first.
  pub fn get_history(&self, count: Option<usize>) -> Vec<&Notification> {
      let iter = self.history.iter().rev();
      match count {
          Some(n) => iter.take(n).collect(),
          None    => iter.collect(),
      }
  }
  ```

      if let crate::Command::History { last } = cmd {
