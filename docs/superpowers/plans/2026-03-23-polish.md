# cooee Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Improve code quality (decompose `build_notification_window`, collapse serde boilerplate) and implement missing features (`image-data`/`image-path` rendering, `replaces_id` in-place replacement, config hot-reload).
**Architecture:** Each task is self-contained and independently testable; Tasks 1–5 are pure feature/refactor work on existing types, while Tasks 6–7 introduce `SharedConfig` (`Arc<RwLock<Config>>`) and a `notify`-based file watcher wired into the existing GTK event-queue pattern. No new public API surface is introduced.
**Tech Stack:** `gtk4` (gdk/glib already present), `zvariant` (bundled with `zbus`), `notify = "6"` (new dependency), `std::sync::{Arc, RwLock}`.

  ```rust
  use crate::notification::ImageData;

  /// Parse the `image-data` hint value (zvariant struct type `(iiibiiay)`).
  /// Returns `None` if the hint is missing or cannot be decoded.
  fn parse_image_data(hints: &HashMap<String, zbus::zvariant::OwnedValue>) -> Option<ImageData> {
      let val = hints.get("image-data")?;
      // OwnedValue derefs to Value; downcast to Structure
      let structure = val.downcast_ref::<zbus::zvariant::Structure>().ok()?;
      let fields = structure.fields();
      if fields.len() < 7 {
          return None;
      }
      let width        = i32::try_from(&fields[0]).ok()?;
      let height       = i32::try_from(&fields[1]).ok()?;
      let rowstride    = i32::try_from(&fields[2]).ok()?;
      let has_alpha    = bool::try_from(&fields[3]).ok()?;
      let bits_per_sample = i32::try_from(&fields[4]).ok()?;
      let n_channels   = i32::try_from(&fields[5]).ok()?;
      // field[6] is `ay` — an array of bytes
      let byte_array = fields[6].downcast_ref::<zbus::zvariant::Array>().ok()?;
      let data: Vec<u8> = byte_array
          .iter()
          .filter_map(|v| u8::try_from(v).ok())
          .collect();
      Some(ImageData { width, height, rowstride, has_alpha, bits_per_sample, n_channels, data })
  }

  /// Parse the `image-path` hint value. Returns `None` if the hint is absent.
  fn parse_image_path(hints: &HashMap<String, zbus::zvariant::OwnedValue>) -> Option<String> {
      let val = hints.get("image-path")?;
      String::try_from(val).ok()
  }
  ```

  ```rust
  fn build_image(notification: &Notification, config: &Config) -> Option<gtk4::Image> {
      if let Some(ref d) = notification.image_data {
          let format = if d.has_alpha {
              gdk::MemoryFormat::R8g8b8a8
          } else {
              gdk::MemoryFormat::R8g8b8
          };
          let texture = gdk::MemoryTexture::new(
              d.width,
              d.height,
              format,
              &gtk4::glib::Bytes::from(&d.data),
              d.rowstride as usize,
          );
          let img = gtk4::Image::from_paintable(Some(&texture));
          img.add_css_class("notification-image");
          img.set_pixel_size(config.general.icon_size * 4);
          return Some(img);
      }

      if let Some(ref p) = notification.image_path {
          let img = if p.starts_with('/') || p.starts_with("file://") {
              gtk4::Image::from_file(crate::config::Config::expand_path(p))
          } else {
              gtk4::Image::from_icon_name(p)
          };
          img.add_css_class("notification-image");
          img.set_pixel_size(config.general.icon_size * 4);
          return Some(img);
      }

      None
  }
  ```

## Implementation Notes

### zvariant `Structure` field extraction

`OwnedValue` implements `Deref<Target = Value>`. To downcast to a `Structure`:

```rust
val.downcast_ref::<zbus::zvariant::Structure>()
```

This returns `Result<&Structure, _>`. Call `.fields()` on the `Structure` to get a `&[Value<'_>]`. Extract scalar fields with `i32::try_from(&fields[n])`, `bool::try_from(&fields[n])`, etc. The byte array field (`ay`) must be downcast to `zbus::zvariant::Array` and iterated.

### GTK `MemoryTexture` rowstride

The `rowstride` parameter to `gdk::MemoryTexture::new` is `usize`, but `ImageData.rowstride` is `i32` (matching the D-Bus type). Cast with `d.rowstride as usize`.

### `position_window` refactor

After Task 6, `position_window` takes an explicit `&Config` snapshot. This avoids holding the `RwLock` guard across GTK calls. All seven `Position` match arms remain unchanged; only the function signature and the way `cfg` is obtained changes.

### `config_path` is a free function

In the codebase, `config_path()` is defined as a free function `pub fn config_path() -> PathBuf` in `src/config.rs`, not as `Config::config_path()`. Use `crate::config::config_path()` in the watcher task.

### Locking discipline

All `SharedConfig` read accesses follow this pattern:

```rust
let snapshot = config.read().unwrap().some_field.clone();
// lock released here
// use snapshot for GTK / async / blocking work
```

Never hold a read or write lock across an `await` point, a `gtk4::glib` callback registration, or a blocking I/O call.
